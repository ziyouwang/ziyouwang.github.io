Java基础——多态

多态是Java的三大特性之一，也是面试常考题，这里来介绍一下什么是多态，如何实现多态，哪些情况下需要用到多态。

一、什么是多态

多态是同一个行为具有多个不同表现形式或形态的能力。

多态的存在有三个前提：

  1. 要有继承关系
  2. 子类要重写（override）父类的方法
  3. 父类引用指向子类对象

子类Cat重写了父类Animal的非静态成员方法an.eat()的输出结果为：猫吃饭
重写了父类的静态方法an.sleep()的输出结果为：动物睡觉
未被重写的方法an.run()的输出结果为：动物奔跑
据此我们可以得出多态成员访问的特点：
成员变量（在类之内、方法之外定义的变量）
编译看左边（父类），运行也看左边（父类）
成员方法（类之内创建的方法）
编译看左边（父类），运行看右边（子类）。动态绑定
静态方法（static修饰的方法）
编译看左边（父类），运行也看左边（父类）。

总结：只有非静态的成员方法才是编译看左边运行看右边。

二、如何实现多态

有三种实现方式：

  1. 接口实现
  2. 继承父类进行方法重写（Override）
  3. 同一个类中进行方法重载（Overload）

三、多态的使用情景
